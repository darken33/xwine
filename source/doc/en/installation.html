2.1. Required Configuration

Required libraries :

  • GTK : http://www.gtk.org
  • Gnome & Gnome-libs : http://developper.gnome.org
  •  libxml2 library : http://rpmfind.net/linux/RPM
  • db1 : http://rpmfind.net/linux/RPM
Note :
If you install XWine from sources, you will need also the header files (includes) of the libraries, especially the gnome-libs-devel package.

Required applications :
You will need also several applications to run Xwine :

  • Wine : XWine is a configuration graphical interface for Wine (it is not the emulator) http://www.winehq.org
  • A web browser : to read the documentation (Konqueror, Netscape, Mozilla, Opera)
  • Wine Utils : utilities for Wine, as notepad, mine, ...(it is not mandatory) http://www.winehq.org
  • WinesetupTk : another configuration graphical interface for Wine (it is not mandatory)

2.2. Installation from sources

The source files are in the  xwine-1.0.tar.gz file , you should first decompress it :
$ tar -xzf xwine-1.0.tar.gz

A xwine-1.0 directory is created, enter in it :
$ cd xwine-1.0_fr

Configure the  XWine installation :
$ ./configure

Compile XWine :
$ make

When the compilation is done, install XWine in the right directory ; you should be root to do it :
$ su
password:
#

Install Xwine using the make install command :
# make install

Then quit the root session :
# exit
$

Now you can run XWine, using the xwine command :
$ xwine

2.3. Installation from RPM file

When you install XWine from RPM file, the whole installation is managed with rpm (dependancies, files, uninstallation). You have not to compile it, so you do not need the header files of the libraries (.h files).

For example, to install XWine from rpm file xwine-1.0-1.i586.rpm, you  have only to type the command (you should be root) :
# rpm -i xwine-1.0.-1.i586.rpm

For the uninstallation, you only type :
# rpm -e xwine
